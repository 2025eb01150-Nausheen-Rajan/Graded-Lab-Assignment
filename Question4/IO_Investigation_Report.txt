Output redirection: 
Command: echo "This is a test message" > output.txt

coder@Ubuntu:~$ echo "This is a test message" > output.txt
coder@Ubuntu:~$ cat output.txt
Output: 
This is a test message

Observation: The output redirection operator (>) redirected the command output into output.txt instead of displaying it on the terminal. The file was created successfully and contained the expected message. This demonstrates how Linux can store command output in files for logging and documentation purposes.

Observation: The command attempted to access a file that does not exist. The error message was redirected to error.txt using file descriptor 2 (stderr), demonstrating error redirection.

Explanation of how Linux manages file I/O: Linux manages input and output using file descriptors. Every process is automatically assigned three standard file descriptors: standard input (0), standard output (1), and standard error (2). Files, devices, sockets, and pipes are all treated as files and are accessed through file descriptors. Redirection operators such as > and 2> allow output and error streams to be redirected to files instead of being displayed on the terminal, making logging, monitoring, and troubleshooting easier.

Conclusion: The investigation demonstrated how Linux manages open files, file descriptors, redirects output streams, and enforces resource limits. These mechanisms provide efficient and flexible file management for applications and system processes.
