Question 1 – Linux Environment Verification

You have joined a company as a Linux support engineer. Before accessing company resources, you must verify your Linux environment and user account details.

Performed the necessary Linux commands and prepared a report containing:

1. Username and groups associated with your account
Command: whoami 
Output: coder
Observation: The whoami command was used to identify the currently logged-in user. The output showed that the active user account is "coder". This confirms the identity of the user currently operating the system. Verifying the username is important before accessing system resources and performing administrative tasks.

Command: groups

coder@Ubuntu:~/Question1$ groups
Output: coder adm cdrom sudo dip plugdev users lpadmin vboxsf
Observation: The groups command displayed all groups associated with the current user account. The user "coder" belongs to multiple groups, including sudo, which provides administrative privileges, and vboxsf, which allows access to VirtualBox shared folders. Group memberships determine the permissions and resources that the user can access on the system. This information is important for managing security and access control in a Linux environment.

2. Current shell being used
Command: echo $SHELL
Output: /bin/bash
Observation: The echo $SHELL command displayed the default shell currently being used by the user. The output showed that Bash (/bin/bash) is the active shell. Bash acts as the command interpreter between the user and the Linux operating system. It is one of the most commonly used shells for Linux administration and scripting.

3. Current working directory
Command: pwd
Output: /home/coder/Question1
Observation: The pwd command displayed the absolute path of the current working directory. The output showed that the user is currently working inside the Question1 directory located in the home folder. This command helps users determine their location within the Linux file system. Knowing the current directory is important when creating, modifying, or accessing files.

4. List of files and directories in your workspace
Command: ls -la
Output: 
total 8
drwxrwxr-x  2 coder coder 4096 Jun 21 18:43 .
drwxr-x--- 11 coder coder 4096 Jun 21 18:43 ..

Observation: The ls -la command displayed all files and directories in the current workspace, including hidden entries. The output showed the current directory (.) and the parent directory (..), along with their permissions, ownership information, and sizes. This command is useful for examining directory contents and verifying file permissions. The workspace was currently empty except for the standard directory entries.
